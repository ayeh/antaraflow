Button

Interactive controls that trigger actions. Use the right variant for the right context.

Variants

Sizes

✓ Do

Use primary for the single most important action on a page.

✗ Don't

Use multiple primary buttons on the same page — only one per view.

Accessibility
  • Icon-only buttons must have aria-label
  • Minimum touch target: 44×44px on mobile
  • disabled attribute for disabled state, not just opacity
  • Focus ring: focus:ring-2 focus:ring-violet-500 focus:ring-offset-2
<button class="inline-flex items-center gap-2 bg-violet-600 hover:bg-violet-700 active:bg-violet-800 text-white px-4 py-2 rounded-lg text-sm font-medium transition-colors focus:outline-none focus:ring-2 focus:ring-violet-500 focus:ring-offset-2 disabled:opacity-40 disabled:cursor-not-allowed">
  Save Meeting
</button>

Input

Text fields for capturing user data. Always pair with a visible label.

Please enter a valid email address.

Email address verified.

✓ Do

Always pair inputs with visible labels (not just placeholder text).

✗ Don't

Use placeholder as the only label — it disappears when typing.

Accessibility
  • Labels must be associated with for/id
  • Use aria-invalid="true" on error state
  • Use aria-describedby for helper text and error messages
<div>
  <label for="meeting-title" class="block text-xs font-medium text-gray-700 mb-1.5">Meeting title</label>
  <input id="meeting-title" type="text" placeholder="Meeting title..."
    class="w-full bg-white border border-gray-300 rounded-lg px-3 py-2 text-sm
           focus:outline-none focus:ring-2 focus:ring-violet-500 focus:border-transparent transition-colors" />
</div>

Textarea

Multi-line text input for longer content like notes and descriptions.

0 / 500
✓ Do

Use resize-none and set an appropriate rows attribute.

✗ Don't

Let textarea resize horizontally — it breaks layout.

Accessibility
  • Same label rules as input — always associate with for/id
  • Use aria-describedby to link the character count to the textarea
<div>
  <div class="flex items-center justify-between mb-1.5">
    <label for="notes" class="text-xs font-medium text-gray-700">Meeting notes</label>
    <span id="notes-count" class="text-xs text-gray-400">0 / 500</span>
  </div>
  <textarea id="notes" rows="4" maxlength="500" aria-describedby="notes-count"
    placeholder="Add notes, context, or pre-reads for attendees..."
    class="w-full border border-gray-300 rounded-lg px-3 py-2 text-sm
           focus:outline-none focus:ring-2 focus:ring-violet-500 focus:border-transparent resize-none"></textarea>
</div>

Toggle

Switch control for immediate on/off settings that take effect without form submission.

✓ Do

Use for immediate on/off settings (no form submit needed).

✗ Don't

Use a toggle where the action requires confirmation — use a button + modal.

Accessibility
  • Add role="switch" and aria-checked for assistive technology
  • Label must describe the setting, not just say "Toggle"
<label class="flex items-center gap-3 cursor-pointer">
  <div class="relative">
    <input type="checkbox" role="switch" class="sr-only peer" />
    <div class="w-11 h-6 bg-gray-200 peer-checked:bg-violet-600 rounded-full transition-colors duration-200"></div>
    <div class="absolute top-0.5 left-0.5 w-5 h-5 bg-white rounded-full shadow-sm transition-transform duration-200 peer-checked:translate-x-5"></div>
  </div>
  <span class="text-sm text-gray-700">Email notifications</span>
</label>

File Upload

Drag-and-drop zone with a click-to-browse fallback for uploading documents and media.

Drop files here or browse

PDF, DOCX, MP3 — up to 50MB

✓ Do

Show allowed file types and size limits in the drop zone.

✗ Don't

Only support drag-and-drop — always include a click-to-browse fallback.

Accessibility
  • The clickable zone must have tabindex="0" and role="button" to be keyboard-focusable
  • Show accepted file types in visible text, not just via accept attribute
<div tabindex="0" role="button" aria-label="Upload files"
  class="border-2 border-dashed border-gray-300 rounded-xl p-10 text-center
         hover:border-violet-400 transition-colors cursor-pointer
         focus:outline-none focus:ring-2 focus:ring-violet-500 focus:ring-offset-2"
  onclick="document.getElementById('file-input').click()"
  onkeydown="if(event.key==='Enter'||event.key===' ')document.getElementById('file-input').click()">
  <input type="file" id="file-input" class="hidden" />
  <p class="text-sm font-medium text-gray-700">Drop files here or <span class="text-violet-600 underline">browse</span></p>
  <p class="text-xs text-gray-400 mt-1.5">PDF, DOCX, MP3 — up to 50MB</p>
</div>

Alert

Inline messages for system feedback. Always contextual to the content around them.

Transcription in progress

AI is processing your recording. This usually takes 2–3 minutes.

Meeting finalized successfully

All attendees have been notified and the minutes are locked.

3 action items overdue

Review and reassign overdue items to keep the project on track.

Failed to save changes

Your changes could not be saved. Please check your connection and try again.

✓ Do

Place alerts inline near the content they relate to, not always at the top of the page.

✗ Don't

Use alerts for every action result — use Toast for transient confirmations.

Accessibility
  • Use role="alert" for errors and important warnings so screen readers announce them immediately.
  • Info/success can use role="status" for polite announcements.
<!-- Info alert (role="status" for polite announcement) -->
<div role="status" class="flex gap-3 bg-blue-50 border border-blue-200 rounded-xl p-4">
  <svg class="w-5 h-5 text-blue-500 shrink-0 mt-0.5" fill="currentColor" viewBox="0 0 20 20">...</svg>
  <div class="flex-1 min-w-0">
    <p class="text-sm font-semibold text-blue-800">Transcription in progress</p>
    <p class="text-xs text-blue-700 mt-0.5">AI is processing your recording.</p>
  </div>
  <button onclick="this.closest('.flex').remove()" class="text-blue-400 hover:text-blue-600 text-lg leading-none">&times;</button>
</div>

<!-- Error alert (role="alert" for immediate announcement) -->
<div role="alert" class="flex gap-3 bg-red-50 border border-red-200 rounded-xl p-4">

    <p class="text-xs text-red-700 mt-0.5">Please check your connection and try again.</p>
  </div>
</div>

Toast

Transient notifications for action confirmations. Auto-dismiss after 4 seconds.

Variants (static preview)

Meeting saved successfully
Failed to delete action item
Unsaved changes will be lost
Transcription ready to review

Live demo

✓ Do

Use toasts for confirmations of completed actions (save, delete, copy).

✗ Don't

Use toasts for errors that require user action — use an inline Alert instead.

Accessibility
  • role="alert" on each toast so screen readers announce it immediately.
  • Keep messages under 80 characters. Never require interaction to dismiss.
function showToast(msg, type) {
  var colors = { success: 'bg-green-600', error: 'bg-red-600', warning: 'bg-amber-500', info: 'bg-blue-600' };
  var t = type || 'info';
  var toast = document.createElement('div');
  toast.className = 'pointer-events-auto flex items-center gap-3 ' + (colors[t] || colors.info)
    + ' text-white px-4 py-3 rounded-xl shadow-lg text-sm font-medium'
    + ' opacity-0 translate-y-2 transition-all duration-300';
  toast.setAttribute('role', 'alert');
  toast.innerHTML = '<span>' + msg + '</span>';
  var container = document.getElementById('toast-container');
  container.appendChild(toast);
  requestAnimationFrame(function() {
    requestAnimationFrame(function() {
      toast.classList.remove('opacity-0', 'translate-y-2');
    });
  });
  setTimeout(function() {
    toast.classList.add('opacity-0', 'translate-y-2');
    setTimeout(function() { if (toast.parentNode) toast.parentNode.removeChild(toast); }, 300);
  }, 4000);
}

✓ Do

Use modals for actions that need explicit confirmation or a focused form.

✗ Don't

✓ Do

Use tooltips for icon-only buttons and keyboard shortcut hints.

✗ Don't

Put essential information only in a tooltip — it's inaccessible on mobile and for keyboard users.

Accessibility
  • Tooltip content must also be in aria-label or aria-describedby.
  • Never put interactive content inside a tooltip.
<!-- CSS-only tooltip (top placement) -->
<div class="tooltip-wrap">
  <button aria-label="Download transcript">
    <svg ...></svg>
  </button>
  <div class="tooltip-tip">Download transcript</div>
</div>

<!-- Right placement -->
<div class="tooltip-wrap">
  <button>Hover me</button>
  <div class="tooltip-tip right">Opens in new tab</div>
</div>

<!-- Bottom placement -->
<div class="tooltip-wrap">
  <button>Hover me</button>
  <div class="tooltip-tip bottom">Delete this item</div>
</div>

✓ Do

Always pair the spinner with a text label like "Saving..." or "Loading..." for context.

✗ Don't

Use a spinner for operations longer than ~5s — show a progress bar with percentage instead.

Accessibility
